# DTF Games Promo Pack — 30 объектов

Пак для проморежима «Срач дня» в «Сходится!».

## Что внутри

- 30 игр, вокруг которых сформировался узнаваемый русскоязычный игровой дискурс.
- Для каждой игры — две стартовые сатирические реплики.
- Дополнительные подсказки после 5-й, 8-й и 9-й попыток.
- Ссылка на поиск материалов DTF, справочный источник и Steam, когда он применим.
- Минимальная резервная карточка ответа для объектов, которых нет в активной ревизии каталога.
- Флаги сложности, узнаваемости и риска сюжетных спойлеров.
- Все «комментарии» написаны специально для игры и не являются цитатами реальных пользователей.

## Почему это совместимо с текущей игрой

Новый `TitleMode` не нужен. Пак остаётся вариантом режима `game`.

Текущий движок уже умеет:

- искать игру в каталоге;
- принимать до десяти попыток;
- сравнивать год, жанры, платформы, разработчика, издателя, Steam, Metacritic, отзывы, цену и возрастной рейтинг;
- передавать `progressiveHints`;
- сравнивать результаты вызова по числу попыток.

Поэтому проморежим можно подключить как `variantKey = "dtf-games-promo-30-v1"` и продолжать использовать обычный `compareGames()`.

## Рекомендуемая механика

1. Сервер выбирает одну запись пакета.
2. `answerRef` разрешается против активной ревизии каталога:
   1. `steamAppId`;
   2. нормализованное название + год;
   3. нормализованное название;
   4. альтернативные названия.
3. До первой попытки показываются две записи `progressiveHints` с `unlockAfterAttempts: 0`.
4. После каждой догадки работает существующее сравнение игровых карточек.
5. После 5-й, 8-й и 9-й ошибки открывается следующая авторская реплика.
6. После победы или десятой ошибки показывается обычная карточка ответа и `result.tagline`.
7. В ссылку вызова добавляется `pack=dtf-games-promo-30-v1`, чтобы друг получил тот же объект.

## Минимальные изменения API

Рекомендуемый запрос старта:

```json
{
  "kind": "daily",
  "mode": "game",
  "period": "all",
  "difficulty": null,
  "packId": "dtf-games-promo-30-v1"
}
```

Дополнительные поля игровой сессии:

```ts
type PromoPrompt = {
  packId: string
  title: string
  subtitle: string
  disclaimer: string
}

type PromoHint = {
  key: string
  unlockAfterAttempts: 0 | 5 | 8 | 9
  type: 'archetype_comment' | 'satirical_context' | 'community_meme' | 'factual_rescue'
  authorArchetype?: string
  text: string
  spoilerRisk: 'low' | 'medium' | 'high'
}
```

В `GameSessionSnapshot` достаточно добавить `promoPrompt`, а существующий `progressiveHints` начать наполнять доступными на текущей попытке записями.

## Разрешение ответов

`fallbackAnswerCard` не является полностью обогащённой производственной карточкой. В ней достаточно данных, чтобы сравнивать год, жанры, платформы, разработчика и издателя. Рейтинги, число отзывов, изображения, цены и Steam-категории следует получить существующим пайплайном обогащения.

В паке:

- 21 объект рассчитан прежде всего на разрешение против существующей базы;
- 9 новых, недавних или нестандартных объектов помечены `resolve_or_import`;
- у Escape from Tarkov официальным годом указан 2025-й, а 2017-й сохранён как `legacyReleaseYears`;
- у Skyrim перечислены App ID оригинала и Special Edition;
- у Heroes III App ID HD Edition оставлен альтернативным, а основное разрешение должно идти по названию и 1999 году.

## Спойлеры

Высокий риск отмечен у:

- The Last of Us Part II;
- Mass Effect 3.

Средний риск отмечен у Death Stranding, Disco Elysium, Pathologic 2 и Silent Hill 2. Для промостатьи можно исключить карточки с `spoilerProfile.level = "high"` или показывать предупреждение.

## Модерация

Перед публикацией нужно:

1. Открыть `sources` каждой записи и проверить, что мем или инфоповод действительно узнаваем на DTF.
2. Убедиться, что текст не совпадает дословно с реальным комментарием.
3. Проверить отсутствие упоминания ответа в подсказках до спасательной подсказки.
4. Прогнать карточку минимум на трёх людях:
   - старт должен давать 3–7 разумных гипотез;
   - после пятой попытки — 2–4;
   - после восьмой — 1–2;
   - девятая подсказка должна почти отдавать ответ.
5. Не использовать имена обычных пользователей DTF без отдельного согласования.

## Редакторская оговорка

Пак передаёт тон игровой части DTF, но не утверждает, что каждая реплика выражает мнение всего сообщества. Это сатирические портреты публичных дискуссий, а не цитатник и не социологическое исследование.
