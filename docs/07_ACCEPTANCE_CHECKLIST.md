# Acceptance Checklist

## MVP считается готовым, если выполнено всё ниже

### Базовая работа

- [ ] Проект запускается через `npm install` и `npm run dev`.
- [ ] Проект собирается через `npm run build` без TypeScript-ошибок.
- [ ] Есть режим `Фильм дня`.
- [ ] Есть режим `Сериал дня`.
- [ ] Есть фильтр периода.
- [ ] Для каждого режима и периода выбирается стабильный тайтл дня по московской дате.
- [ ] После перезагрузки страницы текущий прогресс сохраняется.

### Игровая логика

- [ ] У игрока 10 попыток.
- [ ] Пропуск попытки невозможен.
- [ ] Нельзя отправить тайтл вне базы текущего режима/периода.
- [ ] Нельзя повторно отправить уже использованный тайтл.
- [ ] Победа наступает при выборе правильного тайтла.
- [ ] Поражение наступает после 10 неверных попыток.
- [ ] После каждой попытки появляются плитки-подсказки.
- [ ] Числовые критерии показывают выше/ниже.
- [ ] Частичные совпадения работают для актёров, жанров, стран и студий.
- [ ] Игра не ломается при отсутствии части данных.

### Поиск

- [ ] Поиск работает по русскому названию.
- [ ] Поиск работает по оригинальному названию.
- [ ] Поиск работает по альтернативным названиям.
- [ ] `е` и `ё` считаются совместимыми.
- [ ] Дефисы и лишние пробелы не мешают поиску.
- [ ] Есть обработка базовых опечаток или подключён Fuse.js.
- [ ] В подсказке поиска есть постер, название и год.

### UI

- [ ] Интерфейс mobile-first.
- [ ] Есть главная промо-секция.
- [ ] Есть тёмный ретро-кинотеатральный визуальный стиль.
- [ ] Нет светлой темы.
- [ ] Нет звуков.
- [ ] Есть анимация появления/переворота плиток.
- [ ] Есть экран победы.
- [ ] Есть экран поражения.
- [ ] Есть архив.
- [ ] Есть статистика.

### Шаринг

- [ ] Результат можно скопировать в буфер.
- [ ] Есть ссылка поделиться в Telegram.
- [ ] Есть ссылка поделиться во ВКонтакте.
- [ ] Шаринг не раскрывает правильный ответ.

### Архив

- [ ] Архив показывает прошедшие даты.
- [ ] Старые загадки можно переигрывать.
- [ ] Архивный прогресс хранится отдельно.

### Данные

- [ ] JSON-файлы валидны.
- [ ] Есть минимум тестовых фильмов и сериалов.
- [ ] Каждый тайтл имеет уникальный `id`.
- [ ] Жанры нормализованы на русском.
- [ ] `popularityScore` в диапазоне 0–100.
- [ ] В игре нет падений из-за пустых полей.

### Ограничения MVP

- [ ] Нет backend.
- [ ] Нет авторизации.
- [ ] Нет админки.
- [ ] Нет аналитики.
- [ ] Нет рекламы.
- [ ] Нет PWA.
- [ ] Нет пользовательских загадок.
