# Game Rules

## Режимы

В MVP есть два независимых режима:

- `movie` — фильм дня;
- `series` — сериал дня.

Для каждого режима выбирается отдельный тайтл дня.

## Периоды

Игрок может исключить старые тайтлы через фильтр периода. Фильтр влияет на пул возможных ответов и попыток.

Рекомендуемые периоды:

- `all` — все годы;
- `from_1960` — с 1960 года;
- `from_1980` — с 1980 года;
- `from_1990` — с 1990 года;
- `from_2000` — с 2000 года;
- `from_2010` — с 2010 года;
- `from_2020` — с 2020 года.

Для каждой комбинации `mode + period + Moscow date` должен быть один и тот же ответ у всех игроков.

Пример:

- `movie + all + 2026-07-05` → один фильм;
- `movie + from_2000 + 2026-07-05` → другой фильм из пула после 2000 года;
- `series + from_2000 + 2026-07-05` → отдельный сериал.

Так пользователь может играть в понятном ему периоде, но результат всё равно остаётся ежедневным и общим для всех игроков с тем же фильтром.

## Попытки

- Максимум 10 попыток.
- Пропускать попытку нельзя.
- Игрок может отправить только тайтл из базы текущего режима и периода.
- Если тайтл уже использован в попытке, повторно отправлять его нельзя.

## Победа

Победа наступает, когда пользователь выбирает точный правильный тайтл.

После победы нужно показать:

- название ответа;
- постер;
- количество попыток;
- текущую серию побед;
- общую статистику;
- кнопку копирования результата;
- кнопку поделиться в Telegram;
- кнопку поделиться во ВКонтакте;
- кнопку перейти в архив.

## Поражение

Поражение наступает после 10 неверных попыток.

После поражения нужно показать:

- правильный ответ;
- постер;
- краткую карточку тайтла;
- кнопки шаринга;
- приглашение вернуться завтра.

## Подсказки

После каждой попытки показывается ряд плиток. Каждая плитка сравнивает выбранный тайтл с правильным ответом.

Плитки MVP:

1. Режиссёр / шоураннер.
2. Студия / производство.
3. Год.
4. Популярность у русскоязычной аудитории.
5. Рейтинг Кинопоиска.
6. Рейтинг IMDb.
7. Возрастной рейтинг.
8. Актёры.
9. Жанры.
10. Страна производства.
11. Язык оригинала.
12. Длительность.
13. Бюджет.

Если ряд получается слишком широким для мобильного экрана, плитки должны располагаться горизонтальным скроллом или переноситься в компактную сетку. Mobile UX важнее полного сходства с десктопом.

## Цвета совпадений

Рекомендуемые статусы:

- `match` — точное совпадение;
- `close` — близко;
- `partial` — частичное совпадение;
- `miss` — не совпало;
- `unknown` — данных нет.

Не завязывать логику на конкретные цвета. Цвета задаются на UI-уровне через CSS-переменные.

## Правила сравнения

### Название

Победа только при точном выборе тайтла из базы. Поиск может учитывать опечатки, но отправленная попытка всегда является конкретным объектом базы.

### Год

- `match`: год равен году ответа.
- `close`: разница 1–2 года.
- `miss`: разница больше 2 лет.
- Для неверного года показывать стрелку: ответ старше или новее.

### Рейтинг Кинопоиска

- `match`: разница до 0.1.
- `close`: разница до 0.3.
- `miss`: больше 0.3.
- Показывать стрелку: рейтинг ответа выше или ниже.

### Рейтинг IMDb

Аналогично рейтингу Кинопоиска.

### Популярность

Популярность — внутренний нормализованный показатель, который заменяет кассовые сборы как более понятную для проекта метрику.

Рекомендуется считать `popularityScore` от 0 до 100 на этапе подготовки данных. В него можно включить:

- место в топах Кинопоиска;
- количество оценок Кинопоиска;
- количество голосов IMDb;
- ручной boost для культовых СНГ-тайтлов.

Правила:

- `match`: разница до 3 пунктов;
- `close`: разница до 10 пунктов;
- `miss`: больше 10 пунктов;
- показывать стрелку выше/ниже.

### Возрастной рейтинг

- `match`: одинаковое значение.
- `close`: соседняя категория.
- `miss`: сильное расхождение.

Если рейтинг отсутствует, показывать `нет данных`.

### Актёры

Хранить до 5 ключевых актёров.

- `match`: набор актёров полностью совпадает или выбранный тайтл является правильным ответом.
- `partial`: есть хотя бы один общий актёр.
- `miss`: общих актёров нет.

Порядок актёров не важен.

### Жанры

Хранить до 5 жанров.

- `match`: жанры полностью совпадают.
- `partial`: есть пересечение.
- `miss`: пересечения нет.

### Режиссёр / шоураннер

Для фильмов использовать режиссёров. Для сериалов — showrunner/creator/director, в зависимости от доступных данных.

- `match`: есть точное пересечение по человеку.
- `close`: есть сильная творческая близость, если она явно размечена в данных: общий сценарист, общая франшиза, общий creator. В MVP можно не реализовывать `close`, если данных нет.
- `miss`: совпадений нет.

### Студия

Только точное совпадение или частичное пересечение по студии/производителю. Группировки Disney/Pixar/Marvel не делать автоматически.

### Страна

- `match`: все страны совпадают.
- `partial`: есть пересечение.
- `miss`: пересечения нет.

### Язык оригинала

- `match`: совпадает основной язык оригинала.
- `miss`: не совпадает.

### Длительность

Для фильмов — минуты.
Для сериалов — средняя длительность эпизода или длительность пилотного эпизода, если средняя недоступна.

- `match`: разница до 5 минут.
- `close`: разница до 15 минут.
- `miss`: больше 15 минут.
- Показывать стрелку больше/меньше.

### Бюджет

Если бюджет неизвестен, показывать `нет данных`.

- `match`: разница до 5%.
- `close`: разница до 25%.
- `miss`: больше 25%.
- Показывать стрелку выше/ниже.

## Архив

Архив должен показывать прошедшие дни.

В MVP можно сделать весь архив с момента первого дня, но визуально ограничить список недавними датами и дать кнопку `Показать ещё`.

Старые загадки можно переигрывать. Прогресс по архивным играм хранить отдельно от игры дня.

## Сезонность

Автоматический выбор тайтла должен учитывать сезонность, если в данных есть `seasonTags`.

Примеры:

- Новый год: новогодние фильмы;
- 9 мая: военные фильмы;
- Хэллоуин: хорроры;
- День космонавтики: космос/фантастика.

В MVP можно заложить структуру данных и простую функцию seasonal boost, но не делать сложную админку.
