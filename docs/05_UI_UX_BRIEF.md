# UI / UX Brief

## Общее направление

Тёмный атмосферный сайт в духе ретро-кинотеатра 60-х:

- тёмный зал;
- красный бархат;
- кремовая бумага афиш;
- старые билеты;
- зерно плёнки;
- мягкое свечение вывески;
- табло кинотеатра;
- тревожная ретро-атмосфера без прямого копирования конкретных фильмов, сериалов и постеров.

Можно вдохновляться настроением современных ретро-хорроров и подросточного фантастического кино, но не использовать узнаваемые элементы чужих IP.

## Главная секция

Нужна главная промо-секция перед игрой.

Содержание:

- название `Сеанс`;
- короткий подзаголовок;
- выбор режима: `Фильм дня` / `Сериал дня`;
- выбор периода;
- CTA-кнопка `Начать сеанс`;
- краткое объяснение механики.

Пример текста:

```txt
Сеанс
Угадай фильм или сериал дня по подсказкам. Один день — один тайтл, десять попыток, никаких спойлеров в результате.
```

## Экран игры

Структура:

1. Верхняя панель:
   - логотип;
   - текущий режим;
   - дата сеанса;
   - архив;
   - статистика.

2. Блок управления:
   - переключатель `Фильмы / Сериалы`;
   - фильтр периода;
   - поиск.

3. Игровая область:
   - таблица попыток;
   - строки попыток;
   - плитки-подсказки.

4. Нижняя зона:
   - количество попыток;
   - подсказка по цветам;
   - состояние игры.

## Поиск

Поиск должен быть заметным и удобным на мобильном.

В выпадающем списке показывать:

- постер-миниатюру;
- русское название;
- оригинальное название;
- год;
- страну или режиссёра.

Поиск должен поддерживать:

- русское название;
- оригинальное название;
- альтернативные названия;
- опечатки;
- `е/ё`;
- дефисы/пробелы.

Транслит не нужен.

## Плитки

Плитка должна содержать:

- название критерия;
- значение выбранного тайтла;
- статус совпадения;
- стрелку выше/ниже, если критерий числовой.

На мобильном плитки могут быть компактными. Главное — читаемость.

## Состояния

Нужны состояния:

- старт без попыток;
- активная игра;
- неверная попытка;
- победа;
- поражение;
- архивная игра;
- тайтл уже был использован;
- тайтл не найден;
- данных нет.

## Победа

Экран победы должен быть визуально приятным, но без лишней шумности.

Содержание:

- `Сеанс угадан`;
- постер;
- название;
- год;
- попытки;
- серия побед;
- кнопки шаринга.

## Поражение

Содержание:

- `Сеанс окончен`;
- правильный ответ;
- постер;
- краткая карточка;
- кнопки шаринга;
- приглашение вернуться завтра.

## Архив

Архив показывает даты и статусы:

- дата;
- режим;
- период;
- был ли сыгран;
- победа/поражение;
- количество попыток.

Нажатие на дату открывает архивную игру.

## Статистика

В localStorage хранить:

- всего игр;
- побед;
- поражений;
- процент побед;
- текущая серия;
- лучшая серия;
- распределение побед по попыткам.

Статистика отдельная для фильмов и сериалов, но можно показать общую сводку.

## Шаринг

Результат должен копироваться без спойлера.

Пример:

```txt
Сеанс — Фильм дня
05.07.2026 · с 2000 года
🎬 4/10
🟩⬛⬛🟨⬛🟩⬛⬛
🟩🟨⬛🟨⬛🟩⬛⬛
🟩🟩🟨🟨⬛🟩🟨⬛
🟩🟩🟩🟩🟩🟩🟩🟩
```

Если emoji не хочется использовать в UI, они всё равно допустимы в скопированном результате, как стандарт жанра Wordle-like игр.

## Цветовая система

Примерные CSS-переменные:

```css
:root {
  --bg: #120b0d;
  --surface: #1c1115;
  --surface-2: #2a171d;
  --paper: #f2dfbd;
  --paper-muted: #c7b08d;
  --red: #8f1d2c;
  --red-glow: #c73245;
  --gold: #d6a84f;
  --green: #477b55;
  --yellow: #b98b36;
  --miss: #3a3032;
  --text: #f4ead9;
  --muted: #a99886;
}
```

Агент может адаптировать палитру, но должен сохранить тёмный ретро-кинотеатральный характер.

## Типографика

Рекомендуется использовать системные шрифты для скорости, но можно заложить пары:

- заголовки: выразительный serif/display;
- интерфейс: читаемый sans-serif.

Если агент подключает web-fonts, он должен делать это аккуратно и не ломать производительность.

## Анимации

Нужны:

- плавное появление строки попытки;
- лёгкий flip/reveal плиток;
- мягкое свечение CTA;
- победная анимация без звука.

Не нужны:

- звуки;
- тяжелые canvas/WebGL-эффекты;
- агрессивные мигания.

## Доступность

- Не передавать смысл только цветом: добавить иконки/текст/стрелки.
- Сохранять контраст текста.
- Кнопки не меньше 44px по высоте на мобильном.
- Не блокировать игру, если картинки не загрузились.
