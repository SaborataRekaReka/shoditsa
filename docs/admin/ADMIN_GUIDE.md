# Руководство администратора

Админ-панель открывается по `/admin` в обычной серверной web-сборке. В Yandex Games bundle она не включается. Войдите аккаунтом `breneize@yandex.ru`; если UUID, email или роль не совпадают с allowlist, API вернёт 403 и данные админки не загрузятся.

## Основной рабочий цикл

В левой навигации доступны: обзор, контент, жалобы, пайплайны, пользователи, события, качество, экономика, система и аудит. Поиск в верхней панели одновременно ищет карточки, пользователей и жалобы. Время показывается в `Asia/Almaty`.

### Контент

1. Найдите карточку по названию, alias или ID; при необходимости отфильтруйте режим и публикацию.
2. Откройте drawer. Вкладка «Данные» редактирует payload, «Предпросмотр» показывает карточку, соседние вкладки — жалобы, историю и технические сведения.
3. Сохраните через кнопку или `Ctrl+S`. Изменение попадёт только в общую рабочую версию; активная игра не изменится.
4. При конфликте версии обновите карточку и повторите изменение осознанно. Локальная копия формы сохраняется для восстановления после сбоя вкладки.
5. Запустите проверку рабочей версии. Ошибки блокируют сборку, предупреждения требуют ручной оценки.
6. Запустите сборку revision и дождитесь job. После статуса `ready` отдельно активируйте revision.

Массовые allow/disallow и tags также создают черновые изменения. Новые карточки проходят тот же цикл. Активация атомарна: текущая active revision становится retired, новая — active.

### Жалобы

Очередь слева фильтруется по статусу, причине, режиму, карточке и пользователю. В detail видны snapshot на момент игры, текущая карточка и timeline раунда. Возможные действия: взять в работу, связать с workspace/revision, отметить решённой, отклонить или сделать duplicate. Для итогового решения укажите тип и содержательный комментарий.

### Музыкальный пайплайн

Сначала выберите сценарий и получите estimate. Реальный запуск требует явного подтверждения и `Idempotency-Key`. Worker выполняет существующий enrichment adapter; исходные результаты не публикуются автоматически.

В review сравните before/proposed по каждому полю. Для поля можно принять новое значение, сохранить текущее или задать исправленное вручную. Одобренный item переносится в общую workspace. Только последующая сборка и активация revision меняет игру.

### Пользователи и события

Карточка пользователя объединяет профиль, провайдеры входа, сессии, статистику, кошелёк, жалобы, заметки и timeline. Блокировка требует причины и по умолчанию отзывает сессии. Себя заблокировать, разблокировать или лишить сессий нельзя.

Раздел «События» объединяет игровые старты, попытки, подсказки, жалобы, wallet, auth и client telemetry. Фильтруйте по user/session/game/item/request ID. Большой export выполняется background job.

### Система и аудит

В «Системе» контролируются jobs, health, revisions, будущие materialized загадки и настройки. Failed job можно повторить с новым idempotency key. Stuck означает, что heartbeat старше допустимого интервала; сначала проверьте worker, затем retry. Предыдущую retired revision можно вернуть кнопкой «Откатить» с обязательной причиной.

Изменение daily salt требует текущего значения, нового значения и причины. «Заменить будущую загадку» работает только для будущей московской даты и challenge без начатых сессий. Промокоды и wallet adjustments также являются аудируемыми опасными операциями. В «Аудите» сверяйте actor, action, entity, reason, request ID и before/after.
