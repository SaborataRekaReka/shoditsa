# diagnoses.generated.json

Файл сделан по аналогии с `movies.generated.json`: массив объектов, где каждая запись является самостоятельной игровой сущностью.

Главные отличия от кино:
- `mode` = `diagnosis`;
- вместо режиссёров/актёров/жанров используются медицинские игровые признаки;
- `dataQuality.verified` оставлен `false`, потому что это seed-модель для механики, а не медицински верифицированная база.

Игровые поля:
- `bodySystems`
- `diseaseTypes`
- `course`
- `contagiousness`
- `typicalAgeGroups`
- `localization`
- `keySymptoms`
- `diagnostics`
- `riskFactors`
- `icd10`
- `icdGroup`

Безопасность:
- режим должен показываться как образовательная игра;
- нельзя использовать как самодиагностику;
- перед публикацией нужно сверить записи с клиническими рекомендациями, МКБ-10 и медицинским редактором.
