# Архив проекта

Здесь лежат данные, которые не участвуют в runtime и сборке, но могут быть полезны для аудита или восстановления.

- `legacy-config/` — ранние неиспользуемые справочники.
- `legacy-diagnoses/` — предыдущая форма набора диагнозов и его схема.
- `legacy-scripts/` — superseded одноразовые утилиты и старые pipeline-цепочки.
- `legacy-assets/` — старые неиспользуемые визуальные файлы.
- `reference/` — внешние спецификации и справочные материалы.
- `reports/` — воспроизводимые отчёты проверок и санитизации.
- `samples/` — небольшие эталонные выборки для ручной проверки.
- `local/` — крупные локальные исходники, API evidence и промежуточные AI-результаты. Каталог исключён из Git.

Рабочие данные приложения находятся в `public/data/`. Финальные локальные checkpoints и source-of-truth для повторных pipeline-запусков остаются в `data/`.
